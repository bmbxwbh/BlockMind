## ✨ Особенности проекта

### 🧠 Система памяти — Обучение между сессиями (новое в v3.0)

```
Традиционный подход:  После каждого перезапуска всё забывается, повторные ошибки, повторный расход Token
Подход с памятью:     Три уровня памяти (пространство/пути/стратегии), персистентный JSON, повторное использование между сессиями
```

- **Пространственная память**: Автоматическое обнаружение и запоминание строительных защитных зон, опасных областей, точек ресурсов
- **Память путей**: Кеширование успешных маршрутов, чёрный список неудачных, статистика успешности
- **Стратегическая память**: Успешные операции автоматически сохраняются как многоразовые стратегии, повторное использование без затрат Token
- **Защита построек**: Автоматическое обход строений игрока при навигации — больше никаких разрушений базы

### 🛤️ Интеллектуальная навигация — Путь на основе памяти (новое в v3.0)

```
Традиционный подход:  walk_to(x,y,z) → застревает в стене / пробивает постройку
Интеллектуальная навигация:  запрос памяти → использование кеша → Baritone (с исключением зон) → A* как запасной вариант
```

- **Приоритет кеша**: Пройденные пути используются повторно, нулевые вычисления
- **Интеграция с Baritone**: Самый мощный поисковый движок сообщества, автоматическая прокладка туннелей/мостов/плавание/обход лавы
- **Внедрение защитных зон**: Здания из памяти автоматически добавляются как зоны исключения Baritone
- **Автоматическое обучение**: Результаты каждой навигации автоматически записываются в систему памяти

### 🤖 Архитектура двойного агента — Изоляция чата и операций (новое в v2.0)

```
Главный агент:     Отвечает за чат, персистентный контекст, только распознавание намерений (~50 Token/запрос)
Агент операций:    Отвечает за выполнение, без состояния, новый контекст (<1500 Token/запрос)
```

- **Главный агент**: Поддерживает контекст диалога, распознаёт теги `[TASK:xxx]`
- **Агент операций**: Без состояния, одноразовый, предотвращает взрыв контекста
- **Внедрение памяти**: При принятии решений ИИ автоматически внедряется контекст памяти (защитные зоны зданий, известные маршруты и т.д.)

### 🔌 Архитектура Fabric Mod — Точность и надёжность

- **Нулевой парсинг протокола**: Прямой вызов внутренних API игры
- **13 HTTP-эндпоинтов** + WebSocket-события в реальном времени
- **Опциональная интеграция Baritone**: Продвинутый поиск пути при наличии, базовое линейное перемещение без него

### 🛡️ Пятиуровневая система безопасности

| Уровень | Название | Пример | Стратегия |
|---------|----------|--------|-----------|
| 0 | Полностью безопасно | Перемещение, прыжок | Автоматическое выполнение |
| 1 | Низкий риск | Копать землю, ставить факел | Автоматическое выполнение |
| 2 | Средний риск | Копать руду, атаковать нейтральных существ | Автоматическое выполнение |
| 3 | Высокий risk | Поджечь TNT, залить лавой | Требуется авторизация игрока |
| 4 | Смертельный риск | Поставить командный блок | Запрещено по умолчанию |

### Пример потока данных

**Интеллектуальная навигация на основе памяти:**
```
Игрок говорит "домой"
  → Главный агент распознаёт задачу [TASK:домой]
  → Агент операций сопоставляет навык go_home
  → SmartNavigator запрашивает память:
      ✅ Позиция дома: (65, 64, -120) из памяти игрока
      ✅ Кешированный путь: пройден 3 раза, успешность 100%
      ✅ Зона защиты построек: 30 блоков вокруг базы — запрет на разрушение
      ✅ Опасная зона: (80,12,-50) — лава
  → Навигация Baritone:
      GoalBlock(65, 64, -120)
      + exclusion_zones=[зона защиты базы]
      → Автоматический объезд, ни одна постройка не повреждена
  → По прибытии: кеш пути success_count+1
  → В следующий раз: прямое использование кешированного пути, нулевой расход Token
```

| **Стратегическая память** | Накопленные успешные стратегии, уроки неудач, контекстные теги | ✅ JSON | "При добыче сначала ставь факел, потом копай — максимальная эффективность" |

### Автоматическая защита построек

```python
# Регистрация защитной зоны здания (ИИ запрещено разрушать)
memory.register_building("Главный город", center=(100, 64, 200), radius=30)
# → При навигации автоматически внедряется в Baritone как exclusion_zones
# → тип: "no_break" + "no_place"
# → ИИ не может размещать/разрушать блоки внутри защитной зоны

# Автоматическое обнаружение (сканирование окрестностей каждые 60 секунд)
navigator.auto_detect_and_memorize()
# → Обнаружены последовательные строительные блоки → автоматически регистрируется как защитная зона
# → Обнаружена лава/огонь → автоматически регистрируется как опасная зона
# → Обнаружено скопление руды → автоматически регистрируется как точка ресурсов
```

### Механизм кеширования путей

```python
# Первая навигация: планирование ИИ + выполнение
result = await navigator.goto(100, 64, 200)
# → Путь закеширован: success_count=1, success_rate=100%

# Вторая навигация: прямое использование кеша
result = await navigator.goto(100, 64, 200)
# → Кешированный путь найден, прямое выполнение, нулевые вычисления

# Неудачный путь: автоматическое обучение
# → fail_count >= 3 → автоматически добавляется в чёрный список
# → В следующий раз — перепланирование, старый маршрут не используется
```

### Автоматическое накопление стратегий

```python
# После успешного выполнения операции агентом автоматическая запись
memory.record_strategy(
    task_type="mine",
    description="Сначала факел, потом добыча",
    action_sequence=[...],
    success=True,
    context_tags=["nighttime", "cave"],
)

# При следующей задаче того же типа — автоматический подбор лучшей стратегии
best = memory.get_best_strategy("mine", context_tags=["nighttime"])
# → Возвращает стратегию с наибольшей успешностью
```

## 🛤️ Интеллектуальная навигация

### Процесс навигации

```
SmartNavigator.goto(x, y, z)
  │
  ├── 1. Проверка безопасности
  │     └── Цель в защитной зоне? → Предупреждение, но не отказ
  │
  ├── 2. Запрос кешированных путей
  │     └── Есть надёжный кеш? → Прямое выполнение кешированного пути
  │
  ├── 3. Получение контекста навигации
  │     ├── Зоны исключения (защитные зоны зданий)
  │     ├── Опасные зоны (лава, обрывы)
  │     └── Ссылки на надёжные маршруты
  │
  ├── 4. Поиск пути Baritone (приоритет)
  │     ├── Внедрение exclusion_zones
  │     ├── Автоматическая прокладка туннелей / строительство мостов / плавание
  │     └── Учёт стоимости падения / обход лавы
  │
  ├── 5. Поиск пути A* (запасной вариант)
  │     └── Базовая сетка A* + проверка проходимости блоков
  │
  └── 6. Запись результата
        ├── Успех → cache_path(success=True)
        └── Неудача → cache_path(success=False) + возможное занесение в чёрный список
```

### Интеграция с Baritone

| Характеристика | Baritone | Базовый A* |
|----------------|----------|------------|
| Алгоритм поиска пути | Улучшенный A* + эвристика стоимости | Стандартный A* |
| Прокладка туннелей | ✅ Автоматическое прохождение препятствий | ❌ |
| Строительство мостов | ✅ Режим scaffold | ❌ |
| Плавание | ✅ | ❌ |
| Вертикальное перемещение | ✅ Прыжки/лестницы/лианы | ⚠️ Только 1 блок |
| Обход лавы | ✅ Штраф стоимости | ❌ |
| Стоимость падения | ✅ Учитывается в эвристике | ❌ |
| Зоны исключения | ✅ `exclusionAreas` | ❌ |
| **Защита зданий** | ✅ Внедрение зон `no_break` | ❌ |

### Типы зон исключения

| Тип | Описание | Источник |
|-----|----------|----------|
| `no_break` | Запрет на разрушение блоков | Защитные зоны зданий, база |
| `no_place` | Запрет на размещение блоков | Защитные зоны зданий |
| `avoid` | Полный обход | Опасные зоны (лава и т.д.) |

## 🤖 Архитектура двойного агента

### Зачем нужен двойной агент?

```
Проблема одиночного агента:
  Контекст чата + контекст операций → Взрыв Token (>4000/запрос)
  Ошибка операции загрязняет чат → Плохой опыт диалога
  Каждая операция несёт полную историю чата → Пустая трата

Решение с двойным агентом:
  Главный агент: только чат, скользящее окно 20 сообщений, ~50 Token/запрос
  Агент операций: без состояния, новый контекст, <1500 Token/запрос
```

### Процесс

```
Сообщение игрока
  → Главный агент общается (постоянный контекст)
  → Обнаружен тег [TASK:xxx]
  → Извлечение описания задачи
  → Агент операций выполняет (без состояния):
      ├── Сопоставление навыков
      ├── Внедрение контекста памяти
      ├── L1/L2: Выполнение кешированного навыка
      ├── L3: ИИ заполняет шаблон + выполнение
      └── L4: ИИ полное рассуждение + выполнение
  → Главный агент форматирует ответ → Игрок
```

## 📦 Развёртывание в один клик

### Скачивание

Скачайте с [GitHub Releases](https://github.com/bmbxwbh/BlockMind/releases/latest):

| Файл | Описание |
|------|----------|
| `blockmind-mod-1.0.0.jar` | Fabric Mod (поместите в папку mods/ сервера) |
| `Source code` (zip/tar) | Полный исходный код |

### Запуск в один клик — Linux / macOS

```bash
# Клонирование
git clone https://github.com/bmbxwbh/BlockMind.git
cd BlockMind

# Запуск в один клик (автоустановка зависимостей + MC-сервер + BlockMind + WebUI)
chmod +x start.sh
./start.sh
```

> `start.sh` автоматически: проверяет Python/Java → устанавливает зависимости → сканирует существующий MC-сервер → выбирает версию для установки → запускает всё

### Запуск в один клик — Windows

```cmd
:: Клонирование (или скачайте zip и распакуйте)
git clone https://github.com/bmbxwbh/BlockMind.git
cd BlockMind

:: Установка в один клик
install.bat

:: Запуск в один клик (MC-сервер + BlockMind + WebUI)
start_all.bat
```

> Подробные шаги см. в [Руководстве по развёртыванию на Windows](docs/WINDOWS.md)

### Развёртывание через Docker

```bash
# Получение образа
docker pull ghcr.io/bmbxwbh/blockmind:latest

# Скачивание шаблона конфигурации
wget https://raw.githubusercontent.com/bmbxwbh/BlockMind/main/config.example.yaml -O config.yaml
# Отредактируйте config.yaml, введя конфигурацию вашей модели ИИ

# Запуск
docker run -d \
  --name blockmind \
  -p 19951:19951 \
  -v $(pwd)/config.yaml:/app/config.yaml:ro \
  -v blockmind-data:/data \
  ghcr.io/bmbxwbh/blockmind:latest
```

Или с использованием docker-compose:

```bash
git clone https://github.com/bmbxwbh/BlockMind.git && cd BlockMind
cp config.example.yaml config.yaml
# Отредактируйте config.yaml
docker compose up -d
```

```bash
# Просмотр логов
docker compose logs -f blockmind
# Остановка
docker compose down
```

### Запросы состояния

| Эндпоинт | Метод | Описание |
|-----------|-------|----------|
| `/health` | GET | Проверка работоспособности |

| `/api/inventory` | GET | Информация об инвентаре |
| `/api/entities?radius=32` | GET | Ближайшие сущности |
| `/api/blocks?radius=16` | GET | Ближайшие блоки |

| Эндпоинт | Метод | Описание |
|-----------|-------|----------|

| `/api/place` | POST | Размещение блока |
| `/api/attack` | POST | Атака сущности |
| `/api/eat` | POST | Приём пищи |

| `/api/chat` | POST | Отправка сообщения в чат |

### Планирование маршрута

| Эндпоинт | Метод | Описание |
|-----------|-------|----------|
| `/api/pathfind` | POST | Навигация по маршруту (Baritone/A*) |

- `entity_attack` — Атакован
- `health_low` — Низкое здоровье
- `inventory_full` — Инвентарь полон
- `block_broken` — Блок добыт

## 📝 Система Skill DSL

### Классификация задач

| Уровень | Тип | Пример | Стратегия кеширования |
|---------|-----|--------|----------------------|
| L1 | Фиксированная задача | "домой" | Прямое выполнение |
| L2 | Параметризованная задача | "накопай 10 алмазов" | Кеш с параметрами |
| L3 | Шаблонная задача | "построй укрытие" | Сопоставление шаблонов |
| L4 | Динамическая задача "помоги победить Эндер-дракона" | Рассуждение ИИ |

### Пример YAML-навыка

```yaml
skill_id: mine_diamonds
name: "Добыча алмазов"
level: L2
parameters:
  count: {type: int, default: 10, min: 1, max: 64}
steps:
  - action: pathfind
    target: {y: -59}
    note: "Отправиться к алмазному слою"
  - action: dig_loop
    block: diamond_ore
    count: ${count}
  - action: pathfind
    target: home
    note: "Вернуться на базу"
```

## 🛡️ Система безопасности

| Уровень | Механизм | Описание |
|---------|----------|----------|
| L1 | Оценка риска | Каждое действие оценивается 0-100 |

## ❓ FAQ

**Q: Обязательно ли устанавливать Baritone?**
A: Нет. Baritone — опциональная зависимость. При его отсутствии автоматически используется базовое линейное перемещение A*.

**Q: Где хранятся данные памяти?**
A: В директории `data/memory/` — 5 JSON-файлов, сохраняются между сессиями.

**Q: Как работает защита зданий?**
A: Два способа: ① Ручная регистрация ② Автоматическое обнаружение (сканирование каждые 60 секунд).

**Q: Какие ИИ-провайдеры поддерживаются?**
A: Формат, совместимый с OpenAI (включая DeepSeek/OpenRouter/MiMo и др.) + формат Anthropic.

**Q: Каков размер Docker-образа?**
A: Около 200 МБ, многоэтапная сборка на основе python:3.11-slim.
