## ✨ จุดเด่นของโปรเจกต์

### 🧠 ระบบความทรงจำ — เรียนรู้ข้ามเซสชัน (เพิ่มใน v3.0)

```
วิธีเดิม:    รีสตาร์ททุกครั้งก็ลืมหมด ทำผิดซ้ำๆ สิ้นเปลือง Token ซ้ำๆ
วิธีใหม่:    ความทรงจำ 3 ชั้น (พื้นที่/เส้นทาง/กลยุทธ์) คงถาวรเป็น JSON ใช้ข้ามเซสชันได้
```

- **ความทรงจำเชิงพื้นที่**: ตรวจจับและจดจำเขตอาคารป้องกัน พื้นที่อันตราย จุดทรัพยากรโดยอัตโนมัติ
- **ความทรงจำเชิงเส้นทาง**: แคชเส้นทางสำเร็จ แบล็คลิสต์เส้นทางล้มเหลว สถิติอัตราสำเร็จ
- **ความทรงจำเชิงกลยุทธ์**: ปฏิบัติการสำเร็จจะถูกบันทึกเป็นกลยุทธ์ที่ใช้ซ้ำได้โดยไม่ต้องใช้ Token
- **การป้องกันอาคาร**: หลบหลีกอาคารผู้เล่นโดยอัตโนมัติขณะนำทาง ไม่ต้องกลัวบ้านถูกทำลายอีกต่อไป

### 🛤️ การนำทางอัจฉริยะ — นำทางด้วยความทรงจำ (เพิ่มใน v3.0)

```
วิธีเดิม:    walk_to(x,y,z) → ติดกำแพง / ระเบิดทะลุอาคาร
การนำทางอัจฉริยะ:  ตรวจความทรงจำ → เดินแคช → Baritone(ยกเว้นเขตป้องกัน) → A* สำรอง
```

- **แคชก่อน**: เคยเดินแล้วใช้ซ้ำได้เลย ไม่ต้องคำนวณ
- **รวม Baritone**: เอนจินนำทางที่แข็งแกร่งที่สุดจากชุมชน ขุดทาง/สร้างสะพาน/ว่ายน้ำ/หลบลาวาอัตโนมัติ
- **ฉีดเขตป้องกันอาคาร**: อาคารในความทรงจำถูกฉีดเป็น exclusion zones ของ Baritone โดยอัตโนมัติ
- **เรียนรู้อัตโนมัติ**: ผลลัพธ์การนำทางทุกครั้งถูกบันทึกลงระบบความทรงจำ

### 🤖 สถาปัตยกรรม Dual Agent — แยกแชทกับปฏิบัติการ (เพิ่มใน v2.0)

```
Agent หลัก:   รับผิดชอบแชท ใช้ context ถาวร แค่ระบุเจตนา (~50 Token/ครั้ง)
Agent ปฏิบัติการ: รับผิดชอบดำเนินการ ไม่มี state context ใหม่ทุกครั้ง (<1500 Token/ครั้ง)
```

- **Agent หลัก**: รักษาบริบทการสนทนา ระบุแท็ก `[TASK:xxx]`
- **Agent ปฏิบัติการ**: ไม่มี state ใช้แล้วทิ้ง ป้องกัน context ระเบิด
- **ฉีดความทรงจำ**: ฉีดบริบทความทรงจำอัตโนมัติเมื่อ AI ตัดสินใจ (เขตป้องกันอาคาร เส้นทางที่รู้จัก ฯลฯ)

### 🔌 สถาปัตยกรรม Fabric Mod — แม่นยำและน่าเชื่อถือ

- **ไม่ต้อง parse protocol**: เรียก API ภายในเกมโดยตรง
- **13 HTTP endpoints** + WebSocket เหตุการณ์แบบเรียลไทม์
- **Baritone แบบเลือกได้**: มีก็ใช้นำทางขั้นสูง ไม่มีก็ใช้เส้นตรงพื้นฐาน

### 🛡️ ระบบความปลอดภัย 5 ระดับ

| ระดับ | ชื่อ | ตัวอย่าง | กลยุทธ์ |
|------|------|------|------|
| 0 | ปลอดภัยสมบูรณ์ | เดิน กระโดด | ดำเนินการอัตโนมัติ |
| 1 | ความเสี่ยงต่ำ | ขุดดิน วางคบเพลิง | ดำเนินการอัตโนมัติ |
| 2 | ความเสี่ยงปานกลาง | ขุดแร่ โจมตีมอนสเตอร์ที่เป็นกลาง | ดำเนินการอัตโนมัติ |
| 3 | ความเสี่ยงสูง | จุด TNT วางลาวา | ต้องได้รับอนุญาตจากผู้เล่น |
| 4 | ความเสี่ยงถึงชีวิต | วางบล็อกคำสั่ง | ห้ามโดยค่าเริ่มต้น |

### ตัวอย่างกระแสข้อมูล

**การนำทางอัจฉริยะที่ขับเคลื่อนด้วยความทรงจำ:**
```
ผู้เล่นพูด "กลับบ้าน"
  → Agent หลักระบุงาน [TASK:กลับบ้าน]
  → Agent ปฏิบัติการจับคู่ Skill go_home
  → SmartNavigator ค้นหาความทรงจำ:
      ✅ ตำแหน่งบ้าน: (65, 64, -120) จากความทรงจำผู้เล่น
      ✅ เส้นทางแคช: เดินมาแล้ว 3 ครั้ง อัตราสำเร็จ 100%
      ✅ เขตป้องกันอาคาร: รอบฐาน 30 บล็อก ห้ามทำลาย
      ✅ พื้นที่อันตราย: (80,12,-50) มีลาวา
  → Baritone นำทาง:
      GoalBlock(65, 64, -120)
      + exclusion_zones=[เขตป้องกันฐาน]
      → อ้อมอัตโนมัติ ไม่ทำลายอาคารใดๆ
  → เมื่อถึง: เส้นทางแคช success_count+1
  → ครั้งถัดไปกลับบ้าน: เดินเส้นทางแคชโดยตรง ไม่ต้องใช้ Token
```

## 🧠 ระบบความทรงจำ

### สถาปัตยกรรมความทรงจำ 3 ชั้น

| ชั้น | สิ่งที่เก็บ | คงถาวร | ตัวอย่าง |
|---|---------|--------|------|
| **ความทรงจำเชิงพื้นที่** | เขตป้องกันอาคาร พื้นที่อันตราย จุดทรัพยากร ฐานที่มั่น | ✅ JSON | "พื้นที่ฐาน: (50-100, 60-80, -150--90)" |
| **ความทรงจำเชิงเส้นทาง** | แคชเส้นทางสำเร็จ แบล็คลิสต์เส้นทางล้มเหลว อัตราสำเร็จ | ✅ JSON | "บ้าน→ถ้ำแร่: ผ่าน (70,64,-100) อัตราสำเร็จ 100%" |
| **ความทรงจำเชิงกลยุทธ์** | กลยุทธ์สำเร็จที่สะสม บทเรียนล้มเหลว แท็กบริบท | ✅ JSON | "ขุดแร่ควรวางคบเพลิงก่อนแล้วค่อยขุด มีประสิทธิภาพสูงสุด" |
| **ความทรงจำผู้เล่น** | ตำแหน่งบ้าน เครื่องมือที่ชอบ บันทึกปฏิสัมพันธ์ | ✅ JSON | "บ้านของ Steve อยู่ที่ (100,64,200)" |
| **ความทรงจำโลก** | จุดเกิด จุดปลอดภัย เหตุการณ์สำคัญ | ✅ JSON | "จุดเกิด (0,64,0), รายการจุดปลอดภัย" |

### การป้องกันอาคารอัตโนมัติ

```python
# ลงทะเบียนเขตป้องกันอาคาร (ห้าม AI ทำลาย)
memory.register_building("เมืองหลัก", center=(100, 64, 200), radius=30)
# → ฉีด Baritone exclusion_zones อัตโนมัติขณะนำทาง
# → type: "no_break" + "no_place"
# → AI ไม่สามารถทำลาย/วางบล็อกในเขตป้องกันได้

# ตรวจจับอัตโนมัติ (สแกนรอบๆ ทุก 60 วินาที)
navigator.auto_detect_and_memorize()
# → ตรวจพบบล็อกอาคารต่อเนื่อง → ลงทะเบียนเป็นเขตป้องกันอัตโนมัติ
# → ตรวจพบลาวา/ไฟ → ลงทะเบียนเป็นพื้นที่อันตรายอัตโนมัติ
# → ตรวจพบแร่รวมกัน → ลงทะเบียนเป็นจุดทรัพยากรอัตโนมัติ
```

### กลไกแคชเส้นทาง

```python
# ครั้งแรก: AI วางแผน + ดำเนินการ
result = await navigator.goto(100, 64, 200)
# → แคชเส้นทาง: success_count=1, success_rate=100%

# ครั้งที่สอง: เดินแคชโดยตรง
result = await navigator.goto(100, 64, 200)
# → แคชเส้นทางโดน ดำเนินการโดยตรง ไม่ต้องคำนวณ

# เส้นทางล้มเหลว: เรียนรู้อัตโนมัติ
# → fail_count >= 3 → เพิ่มลงแบล็คลิสต์อัตโนมัติ
# → ครั้งถัดไปวางแผนใหม่ ไม่เดินทางเดิม
```

### การสะสมกลยุทธ์อัตโนมัติ

```python
# Agent ปฏิบัติการบันทึกอัตโนมัติหลังดำเนินการสำเร็จ
memory.record_strategy(
    task_type="mine",
    description="วางคบเพลิงก่อนแล้วค่อยขุดแร่",
    action_sequence=[...],
    success=True,
    context_tags=["nighttime", "cave"],
)

# ครั้งถัดไปจับคู่กลยุทธ์ที่ดีที่สุดสำหรับประเภทงานเดียวกันอัตโนมัติ
best = memory.get_best_strategy("mine", context_tags=["nighttime"])
# → คืนกลยุทธ์ที่มีอัตราสำเร็จสูงสุด
```


### ขั้นตอนการนำทาง

```

  │
  ├── 1. ตรวจสอบความปลอดภัย
  │     └── จุดหมายอยู่ในเขตป้องกัน? → เตือนแต่ไม่ปฏิเสธ
  │
  ├── 2. ค้นหาเส้นทางแคช
  │     └── มีแคชที่เชื่อถือได้? → ดำเนินการเส้นทางแคชโดยตรง
  │
  ├── 3. ดึงบริบทการนำทาง
  │     ├── เขตยกเว้น (เขตป้องกันอาคาร)
  │     ├── พื้นที่อันตราย (ลาวา หน้าผา)
  │     └── อ้างอิงเส้นทางที่เชื่อถือได้
  │
  ├── 4. Baritone นำทาง (ลำดับแรก)
  │     ├── ฉีด exclusion_zones
  │     ├── ขุดทาง / สร้างสะพาน / ว่ายน้ำ อัตโนมัติ
  │     └── ค่าตก / การหลบลาวา
  │
  ├── 5. A* นำทาง (สำรอง)
  │     └── A* กริดพื้นฐาน + ตรวจสอบการเดินผ่านบล็อก
  │
  └── 6. บันทึกผลลัพธ์
        ├── สำเร็จ → cache_path(success=True)
        └── ล้มเหลว → cache_path(success=False) + อาจถูกแบล็คลิสต์
```

### การรวม Baritone

| คุณสมบัติ | Baritone | A* พื้นฐาน |
|------|----------|---------|
| อัลกอริทึมนำทาง | A* ปรับปรุง + ค่าใช้จ่ายแบบ heuristic | A* มาตรฐาน |
| ขุดทาง | ✅ ขุดทะลุสิ่งกีดขวางอัตโนมัติ | ❌ |
| สร้างสะพาน | ✅ โหมด scaffold | ❌ |
| ว่ายน้ำ | ✅ | ❌ |
| เคลื่อนที่แนวตั้ง | ✅ กระโดด/บันได/เถาวัลย์ | ⚠️ แค่ 1 บล็อก |
| หลบลาวา | ✅ ค่าปรับ | ❌ |
| ค่าตก | ✅ คำนวณใน heuristic | ❌ |
| เขตยกเว้น | ✅ `exclusionAreas` | ❌ |
| **การป้องกันอาคาร** | ✅ ฉีดเขต `no_break` | ❌ |

### ประเภทเขตยกเว้น

| ประเภท | คำอธิบาย | แหล่งที่มา |
|------|------|------|
| `no_break` | ห้ามทำลายบล็อก | เขตป้องกันอาคาร ฐานที่มั่น |
| `no_place` | ห้ามวางบล็อก | เขตป้องกันอาคาร |
| `avoid` | หลีกเลี่ยงทั้งหมด | พื้นที่อันตราย (ลาวา ฯลฯ) |

## 🤖 สถาปัตยกรรม Dual Agent

### ทำไมต้องมี Dual Agent?

```
ปัญหาของ Single Agent:
  context แชท + context ปฏิบัติการ → Token ระเบิด (>4000/ครั้ง)
  ปฏิบัติการล้มเหลวปนแชท → ประสบการณ์สนทนาแย่
  ทุกครั้งต้องแบกประวัติแชททั้งหมด → สิ้นเปลือง

วิธี Dual Agent:
  Agent หลัก: แค่แชท หน้าต่างเลื่อน 20 ข้อความ ~50 Token/ครั้ง
  Agent ปฏิบัติการ: ไม่มี state context ใหม่ทุกครั้ง <1500 Token/ครั้ง
```

### ขั้นตอน

```
ข้อความผู้เล่น
  → Agent หลักสนทนา (context ถาวร)
  → ระบุแท็ก [TASK:xxx]
  → แยกรายละเอียดงาน
  → Agent ปฏิบัติการดำเนินการ (ไม่มี state):
      ├── ค้นหา Skill ที่จับคู่
      ├── ฉีดบริบทความทรงจำ
      ├── L1/L2: ดำเนินการ Skill แคช
      ├── L3: AI เติมเทมเพลต + ดำเนินการ
      └── L4: AI ให้เหตุผลเต็มรูปแบบ + ดำเนินการ
  → Agent หลักจัดรูปแบบคำตอบ → ผู้เล่น
```

## 📦 ติดตั้งแบบ One-Click

### ดาวน์โหลด

ดาวน์โหลดจาก [GitHub Releases](https://github.com/bmbxwbh/BlockMind/releases/latest):

| ไฟล์ | คำอธิบาย |
|------|------|
| `blockmind-mod-1.0.0.jar` | Fabric Mod (ใส่ในโฟลเดอร์ mods/ ของเซิร์ฟเวอร์) |
| `Source code` (zip/tar) | ซอร์สโค้ดทั้งหมด |

### เริ่มต้นแบบ One-Click สำหรับ Linux / macOS

```bash
# โคลน
git clone https://github.com/bmbxwbh/BlockMind.git
cd BlockMind

# เริ่มต้นแบบ One-Click (ติดตั้ง dependencies + เซิร์ฟเวอร์ MC + BlockMind + WebUI อัตโนมัติ)
chmod +x start.sh
./start.sh
```

> `start.sh` ทำงานอัตโนมัติ: ตรวจ Python/Java → ติดตั้ง dependencies → สแกนเซิร์ฟเวอร์ MC ที่มีอยู่ → เลือกเวอร์ชันติดตั้ง → เริ่มต้นทั้งหมด

### เริ่มต้นแบบ One-Click สำหรับ Windows

```cmd
:: โคลน (หรือดาวน์โหลด zip แล้วแตกไฟล์)
git clone https://github.com/bmbxwbh/BlockMind.git
cd BlockMind

:: ติดตั้งแบบ One-Click
install.bat

:: เริ่มต้นแบบ One-Click (เซิร์ฟเวอร์ MC + BlockMind + WebUI)
start_all.bat
```

> ดูขั้นตอนละเอียดได้ที่ [คู่มือการ Deploy สำหรับ Windows](docs/WINDOWS.md)

### Deploy ด้วย Docker

```bash
# ดึงอิมเมจ
docker pull ghcr.io/bmbxwbh/blockmind:latest

# ดาวน์โหลดเทมเพลต config
wget https://raw.githubusercontent.com/bmbxwbh/BlockMind/main/config.example.yaml -O config.yaml
# แก้ไข config.yaml ใส่ค่า AI model ของคุณ

# เริ่มต้น
docker run -d \
  --name blockmind \
  -p 19951:19951 \
  -v $(pwd)/config.yaml:/app/config.yaml:ro \
  -v blockmind-data:/data \
  ghcr.io/bmbxwbh/blockmind:latest
```

หรือใช้ docker-compose:

```bash
git clone https://github.com/bmbxwbh/BlockMind.git && cd BlockMind
cp config.example.yaml config.yaml
# แก้ไข config.yaml
docker compose up -d
```

```bash
# ดู logs
docker compose logs -f blockmind
# หยุด
docker compose down
```

### การตั้งค่า

แก้ไข `config.yaml`:

```yaml
ai:
  main_agent:
    provider: "openai"          # openai หรือ anthropic
    api_key: "sk-your-key"
    model: "gpt-4o"             # ชื่อโมเดลของคุณ
    base_url: ""                # URL API แบบกำหนดเอง (ถ้ามี)

webui:
  enabled: true
  port: 19951
  auth:
    password: "your-password"   # รหัสผ่านเข้าสู่ระบบ WebUI
```

หลังเริ่มต้น เข้า `http://localhost:19951` เพื่อเปิดแผงควบคุม

```yaml
skill_id: mine_diamonds
name: "ขุดเพชร"
level: L2
parameters:

steps:
  - action: pathfind
    target: {y: -59}
    note: "ไปยังชั้นเพชร"
  - action: dig_loop
    block: diamond_ore
    count: ${count}
  - action: pathfind
    target: home
```

## ❓ FAQ

**Q: ต้องติดตั้ง Baritone ไหม?**
A: ไม่จำเป็น Baritone เป็น dependency เสริม ไม่มีก็จะสำรองกลับไปใช้ A* พื้นฐานแบบเส้นตรง

**Q: ข้อมูลความทรงจำเก็บไว้ที่ไหน?**
A: ในโฟลเดอร์ `data/memory/` มี 5 ไฟล์ JSON เก็บข้ามเซสชันได้

**Q: การป้องกันอาคารทำงานอย่างไร?**
A: สองวิธี: ① ลงทะเบียนด้วยตนเอง ② ตรวจจับอัตโนมัติ (สแกนทุก 60 วินาที)

**Q: รองรับ AI provider อะไรบ้าง?**
A: รูปแบบที่เข้ากันได้กับ OpenAI (รวมถึง DeepSeek/OpenRouter/MiMo ฯลฯ) + รูปแบบ Anthropic

**Q: อิมเมจ Docker ใหญ่แค่ไหน?**
A: ประมาณ 200MB สร้างแบบ multi-stage จาก python:3.11-slim
