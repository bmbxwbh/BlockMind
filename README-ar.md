## ✨ مميزات المشروع

### 🧠 نظام الذاكرة — التعلم عبر الجلسات (جديد في v3.0)

```
الطريقة التقليدية:  كل إعادة تشغيل تنسى كل شيء، أخطاء متكررة، استهلاك Token متكر
طريقة الذاكرة:    ذاكرة مكانية/مسارات/استراتيجيات من ثلاث طبقات، JSON دائم، إعادة استخدام عبر الجلسات
```

- **الذاكرة المكانية**: كشف تلقائي وحفظ مناطق الحماية المبنية، والمناطق الخطرة، ونقاط الموارد
- **ذاكرة المسارات**: تخزين مؤقت للمسارات الناجحة، وقائمة سوداء للمسارات الفاشلة، إحصائيات نسبة النجاح
- **ذاكرة الاستراتيجيات**: العمليات الناجحة تتحول تلقائيًا إلى استراتيجيات قابلة لإعادة استخدام، إعادة استخدام بـ Zero Token
- **حماية المباني**: تجنب المباني المبنية تلقائيًا أثناء التنقل، لا مخاوف من تدمير المنازل

### 🛤️ تنقل ذكي — بحث مسارات مدفوع بالذاكرة (جديد في v3.0)

```
الطريقة التقليدية:  walk_to(x,y,z) → يعلق في الجدار / يفجر المباني
التنقل الذكي:      استعلام ذاكرة → مسار مخزن → Baritone (استثناء مناطق الحماية) → A* احتياطي
```

- **أولوية التخزين المؤقت**: المسارات المُسلكة مسبقًا تُعاد استخدامها مباشرة، بدون حساب
- **تكامل Baritone**: أقوى محرك بحث مسارات في المجتمع، حفر/بناء جسور/سباحة/تجنب الحمم تلقائيًا
- **حقن مناطق حماية المباني**: المباني المسجلة في الذاكرة تُحقن تلقائيًا كمناطق استبعاد في Baritone
- **تعلم تلقائي**: كل نتيجة تنقل تُسجل تلقائيًا في نظام الذاكرة

### 🤖 بنية الوكيل المزدوج — فصل المحادثة عن التنفيذ (جديد في v2.0)

```
الوكيل الرئيسي:    مسؤول عن المحادثة، سياق دائم، فقط تحديد النوايا (~50 Token/مرة)
وكيل التنفيذ:     مسؤول عن التنفيذ، بلا حالة، سياق جديد (<1500 Token/مرة)
```

- **الوكيل الرئيسي**: يحافظ على سياق المحادثة، يتعرف على وسوم `[TASK:xxx]`
- **وكيل التنفيذ**: بلا حالة، يُستخدم ويُتخلص منه، لتجنب انفجار السياق
- **حقن الذاكرة**: حقن سياق الذاكرة تلقائيًا عند اتخاذ قرارات الذكاء الاصطناعي (مناطق حماية المباني، المسارات المعروفة، إلخ)

### 🔌 بنية Fabric Mod — دقيقة وموثوقة

- **صفر تحليل بروتوكول**: استدعاء مباشر لواجهات اللعبة الداخلية
- **13 نقطة نهاية HTTP** + أحداث WebSocket في الوقت الفعلي
- **تكامل Baritone اختياري**: إن وُجد فتنقل متقدم، وإلا فتنقل خطي أساسي

### 🛡️ نظام أمان من خمس مستويات

| المستوى | الاسم | المثال | الاستراتيجية |
|---------|-------|--------|-------------|
| 0 | آمن تمامًا | حركة، قفز | تنفيذ تلقائي |
| 1 | مخاطر منخفضة | حفر تراب، وضع مشعل | تنفيذ تلقائي |
| 2 | مخاطر متوسطة | حفر خام، مهاجمة كائنات محايدة | تنفيذ تلقائي |
| 3 | مخاطر عالية | إشعال TNT، وضع حمم | يتطلب إذن اللاعب |
| 4 | مخاطر قاتلة | وضع مكامر أوامر | محظور افتراضيًا |

### مثال على تدفق البيانات

**تنقل ذكي مدفوع بالذاكرة:**
```
اللاعب يقول "العودة للمنزل"
  → الوكيل الرئيسي يتعرف على المهمة [TASK:العودة للمنزل]
  → وكيل التنفيذ يطابق Skill go_home
  → SmartNavigator يستعلم الذاكرة:
      ✅ موقع المنزل: (65, 64, -120) من ذاكرة اللاعبين
      ✅ مسار مخزن: مُسلك 3 مرات، نسبة نجاح 100%
      ✅ منطقة حماية مباني: 30 بلوك حول القاعدة ممنوع التدمير
      ✅ منطقة خطرة: (80,12,-50) يوجد حمم
  → تنقل Baritone:
      GoalBlock(65, 64, -120)
      + exclusion_zones=[منطقة حماية القاعدة]
      → يلتف تلقائيًا، لا يدمر أي مبنى
  → بعد الوصول: success_count+1 للمسار المخزن
  → المرة القادمة: يستخدم المسار المخزن مباشرة، استهلاك Zero Token
```

## 🧠 نظام الذاكرة

### بنية الذاكرة الثلاثية

| الطبقة | المحتوى المخزن | دائم | مثال |
|--------|----------------|------|------|
| **ذاكرة مكانية** | مناطق حماية المباني، مناطق خطرة، نقاط موارد، قاعدة | ✅ JSON | "نطاق القاعدة: (50-100, 60-80, -150--90)" |
| **ذاكرة مسارات** | تخزين مؤقت لمسارات ناجحة، قائمة سوداء لمسارات فاشلة، نسبة نجاح | ✅ JSON | "المنزل→كهف التعدين: عبر (70,64,-100) نسبة نجاح 100%" |
| **ذاكرة استراتيجيات** | ترسيخ استراتيجيات ناجحة، دروس فشل، وسوم سياق | ✅ JSON | "عند التعدين ضع مشعلًا أولًا ثم احفروا، الأعلى كفاءة" |
| **ذاكرة لاعبين** | موقع المنزل، أدوات مفضلة، سجل تفاعل | ✅ JSON | "منزل Steve في (100,64,200)" |
| **ذاكرة عالم** | نقطة بداية، نقاط أمان، أحداث مهمة | ✅ JSON | "نقطة البداية (0,64,0)، قائمة نقاط الأمان" |

### الحماية التلقائية للمباني

```python
# تسجيل منطقة حماية مبنى (يمنع الذكاء الاصطناعي من التدمير)
memory.register_building("المدينة الرئيسية", center=(100, 64, 200), radius=30)
# → تُحقن تلقائيًا كـ exclusion_zones في Baritone أثناء التنقل
# → type: "no_break" + "no_place"
# → الذكاء الاصطناعي لا يستطيع تدمير/وضع بلوكات داخل منطقة الحماية

# كشف تلقائي (مسح كل 60 ثانية)
navigator.auto_detect_and_memorize()
# → كشف بلوكات بناء متتالية → تسجيل تلقائي كمنطقة حماية
# → كشف حمم/نار → تسجيل تلقائي كمنطقة خطرة
# → كشف تجمع خامات → تسجيل تلقائي كنقطة موارد
```

### آلية تخزين المسارات مؤقتًا

```python
# أول مرة: تخطيط الذكاء الاصطناعي + تنفيذ
result = await navigator.goto(100, 64, 200)
# → تخزين مؤقت: success_count=1, success_rate=100%

# المرة الثانية: استخدام التخزين المؤقت مباشرة
result = await navigator.goto(100, 64, 200)
# → إصابة التخزين المؤقت، تنفيذ مباشر، صفر حسابات

# المسار الفاشل: تعلم تلقائي
# → fail_count >= 3 → إضافة تلقائية للقائمة السوداء
# → إعادة تخطيط في المرة القادمة، لا يسلك المسار القديم
```

### ترسيخ الاستراتيجيات تلقائيًا

```python
# تسجيل تلقائي بعد نجاح تنفيذ وكيل التنفيذ
memory.record_strategy(
    task_type="mine",
    description="ضع مشعلًا أولًا ثم احفروا",
    action_sequence=[...],
    success=True,
    context_tags=["nighttime", "cave"],
)

# مطابقة أفضل استراتيجية تلقائيًا لنفس نوع المهمة
best = memory.get_best_strategy("mine", context_tags=["nighttime"])
# → يُعيد الاستراتيجية ذات أعلى نسبة نجاح
```

## 🛤️ تنقل ذكي

### سير عملية التنقل

```
SmartNavigator.goto(x, y, z)
  │
  ├── 1. فحص أمان
  │     └── الهدف داخل منطقة حماية؟ → تحذير لكن لا رفض
  │
  ├── 2. استعلام المسار المخزن
  │     └── يوجد تخزين مؤقت موثوق؟ → تنفيذ المسار المخزن مباشرة
  │
  ├── 3. الحصول على سياق التنقل
  │     ├── مناطق مستبعدة (مناطق حماية المباني)
  │     ├── مناطق خطرة (حمم، منحدرات)
  │     └── مرجع مسار موثوق
  │
  ├── 4. بحث مسار Baritone (أولوية)
  │     ├── حقن exclusion_zones
  │     ├── حفر تلقائي / بناء جسور / سباحة
  │     └── تكلفة السقوط / تجنب الحمم
  │
  ├── 5. بحث مسار A* (احتياطي)
  │     └── A* شبكي أساسي + تقييم إمكانية عبور البلوكات
  │
  └── 6. تسجيل النتيجة
        ├── نجاح → cache_path(success=True)
        └── فشل → cache_path(success=False) + possibly blacklist
```

### تكامل Baritone

| الميزة | Baritone | A* أساسي |
|--------|----------|----------|
| خوارزمية البحث | A* محسّن + تقدير تكلفة | A* قياسي |
| حفر ممرات | ✅ حفر تلقائي عبر العوائق | ❌ |
| بناء جسور | ✅ وضع scaffold | ❌ |
| سباحة | ✅ | ❌ |
| حركة عمودية | ✅ قفز/سلالم/كروم | ⚠️ بلوك واحد فقط |
| تجنب الحمم | ✅ عقوبة تكلفة | ❌ |
| تكلفة السقوط | ✅ تُحسب في دالة التقدير | ❌ |
| مناطق مستبعدة | ✅ `exclusionAreas` | ❌ |
| **حماية المباني** | ✅ حقن مناطق `no_break` | ❌ |

### أنواع المناطق المستبعدة

| النوع | الوصف | المصدر |
|-------|-------|--------|
| `no_break` | منع تدمير البلوكات | مناطق حماية المباني، القاعدة |
| `no_place` | منع وضع البلوكات | مناطق حماية المباني |
| `avoid` | تجنب كامل | مناطق خطرة (حمم، إلخ) |

## 🤖 بنية الوكيل المزدوج

### لماذا نحتاج وكيلًا مزدوجًا؟

```
مشكلة الوكيل الواحد:
  سياق محادثة + سياق تنفيذ → انفجار Token (>4000/مرة)
  فشل التنفيذ يُلوث المحادثة → تجربة محادثة سيئة
  كل عملية تنفيذ تحمل كامل تاريخ المحادثة → هدر

حل الوكيل المزدوج:
  الوكيل الرئيسي: محادثة فقط، نافذة منزلقة 20 رسالة، ~50 Token/مرة
  وكيل التنفيذ: بلا حالة، سياق جديد تمامًا، <1500 Token/مرة
```

### السير

```
رسالة اللاعب
  → الوكيل الرئيسي يُحادث (سياق دائم)
  → يتعرف على وسم [TASK:xxx]
  → استخراج وصف المهمة
  → وكيل التنفيذ ينفذ (بلا حالة):
      ├── مطابقة Skill
      ├── حقن سياق الذاكرة
      ├── L1/L2: تنفيذ Skill مخزن
      ├── L3: ملء قالب بالذكاء الاصطناعي + تنفيذ
      └── L4: استنتاج كامل بالذكاء الاصطناعي + تنفيذ
  → الوكيل الرئيسي يُنسّق الرد → اللاعب
```

## 📝 نظام Skill DSL

### تصنيف المهام

| المستوى | النوع | المثال | استراتيجية التخزين المؤقت |
|---------|-------|--------|-------------------------|
| L1 | مهمة ثابتة | "العودة للمنزل" | تنفيذ مباشر |
| L2 | مهمة مُعلَّمة | "احفر 10 ألماسات" | تخزين مؤقت بالمعاملات |
| L3 | مهمة قالبية | "ابنِ مأوى" | مطابقة قوالب |
| L4 | مهمة ديناميكية | "ساعدني في هزيمة Ender Dragon" | استنتاج بالذكاء الاصطناعي |

### مثال Skill YAML

```yaml
skill_id: mine_diamonds
name: "حفر ألماسات"
level: L2
parameters:
  count: {type: int, default: 10, min: 1, max: 64}
steps:
  - action: pathfind
    target: {y: -59}
    note: "الذهاب إلى طبقة الألماس"
  - action: dig_loop
    block: diamond_ore
    count: ${count}
  - action: pathfind
    target: home
    note: "العودة إلى القاعدة"
```

## ❓ الأسئلة الشائعة

**س: هل يجب تثبيت Baritone؟**
ج: لا. Baritone تبعية اختيارية، عند غيابه يتم الرجوع تلقائيًا إلى A* أساسي للحركة الخطية.

**س: أين تُخزَّن بيانات الذاكرة؟**
ج: في مجلد `data/memory/`، 5 ملفات JSON، تبقى عبر الجلسات.

**س: كيف تعمل حماية المباني؟**
ج: بطريقتين: ① تسجيل يدوي ② كشف تلقائي (مسح كل 60 ثانية).

**س: ما مزودي الذكاء الاصطناعي المدعومين؟**
ج: صيغة متوافقة مع OpenAI (بما في ذلك DeepSeek/OpenRouter/MiMo، إلخ) + صيغة Anthropic.

**س: ما حجم صورة Docker؟**
ج: حوالي 200MB، مبنية بتقنية متعددة المراحل باستخدام python:3.11-slim.
