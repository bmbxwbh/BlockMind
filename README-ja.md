## ✨ プロジェクトの特長

### 🧠 記憶システム — セッション間学習（v3.0 新機能）

```
従来方式：  再起動のたびに全て忘却、同じミスを繰り返し、Token を浪費
記憶方式：  空間/パス/戦略の3層記憶、JSON 永続化、セッション間で再利用
```

- **空間記憶**：建築保護区域、危険エリア、資源ポイントを自動検知・記憶
- **パス記憶**：成功パスをキャッシュ、失敗パスをブラックリスト化、成功率を統計
- **戦略記憶**：成功操作を自動的に再利用可能な戦略として蓄積、Token 消費ゼロで再利用
- **建築保護**：ナビゲーション時にプレイヤーの建築物を自動回避、建造物破壊の心配なし

### 🛤️ インテリジェントナビゲーション — 記憶駆動の経路探索（v3.0 新機能）

```
従来方式：  walk_to(x,y,z) → 壁に詰まる / 建築物を破壊
インテリジェント：  記憶を参照 → キャッシュを利用 → Baritone（保護区域除外）→ A* フォールバック
```

- **キャッシュ優先**：通った道はそのまま再利用、計算不要
- **Baritone 統合**：コミュニティ最強の経路探索エンジン、自動掘削/架橋/泳行/溶岩回避
- **建築保護区域注入**：記憶された建築物を自動的に Baritone の除外区域として登録
- **自動学習**：各ナビゲーション結果を記憶システムに自動記録

### 🤖 デュアル Agent アーキテクチャ — チャットと操作の分離（v2.0 新機能）

```
メイン Agent：  チャット担当、永続コンテキスト、意図識別のみ（約50 Token/回）
操作 Agent：    実行担当、ステートレス、毎回全新的コンテキスト（<1500 Token/回）
```

- **メイン Agent**：会話コンテキストを維持し、`[TASK:xxx]` タグを識別
- **操作 Agent**：ステートレス、使用後破棄でコンテキスト爆発を回避
- **記憶注入**：AI 意思決定時に記憶コンテキスト（建築保護区域、既知パス等）を自動注入

### 🔌 Fabric Mod アーキテクチャ — 精密かつ信頼性が高い

- **プロトコル解析ゼロ**：ゲーム内部 API を直接呼び出し
- **13 の HTTP エンドポイント** + WebSocket リアルタイムイベント
- **Baritone オプション統合**：あれば高度な経路探索、なければ基本直線移動

### 🛡️ 5段階セキュリティ体系

| レベル | 名称 | 例 | 戦略 |
|--------|------|------|------|
| 0 | 完全安全 | 移動、ジャンプ | 自動実行 |
| 1 | 低リスク | 土を掘る、松明を置く | 自動実行 |
| 2 | 中リスク | 鉱石を採掘、中立MOBを攻撃 | 自動実行 |
| 3 | 高リスク | TNT点火、溶岩を置く | プレイヤー承認が必要 |
| 4 | 致命的リスク | コマンドブロックを置く | デフォルトで禁止 |

## 🧠 記憶システム

### 3層記憶アーキテクチャ

| 層 | 保存内容 | 永続化 | 例 |
|---|---------|--------|------|
| **空間記憶** | 建築保護区域、危険エリア、資源ポイント、基地 | ✅ JSON | "基地範囲: (50-100, 60-80, -150--90)" |
| **パス記憶** | 成功パスキャッシュ、失敗パスブラックリスト、成功率 | ✅ JSON | "家→鉱山: (70,64,-100)経由 成功率100%" |
| **戦略記憶** | 成功戦略の蓄積、失敗教訓、コンテキストタグ | ✅ JSON | "採掘時はまず松明を置いてから掘る、効率最高" |
| **プレイヤー記憶** | 家の位置、好みのツール、交流記録 | ✅ JSON | "Steve の家は (100,64,200)" |
| **ワールド記憶** | スポーン地点、安全地点、重要なイベント | ✅ JSON | "スポーン地点 (0,64,0)、安全地点リスト" |

### 自動建築保護

```python
# 建築保護区域を登録（AI の破壊を禁止）
memory.register_building("主城", center=(100, 64, 200), radius=30)
# → ナビゲーション時に Baritone exclusion_zones として自動注入
# → type: "no_break" + "no_place"
# → AI は保護区域内でブロックの破壊/設置が不可能

# 自動検知（60秒ごとに周囲をスキャン）
navigator.auto_detect_and_memorize()
# → 連続する建築ブロックを検知 → 自動的に保護区域として登録
# → 溶岩/炎を検知 → 自動的に危険エリアとして登録
# → 鉱石の集中を検知 → 自動的に資源ポイントとして登録
```

### パスキャッシュメカニズム

```python
# 初回ナビゲーション：AI が計画 + 実行
result = await navigator.goto(100, 64, 200)
# → パスキャッシュ: success_count=1, success_rate=100%

# 2回目のナビゲーション：キャッシュを直通利用
result = await navigator.goto(100, 64, 200)
# → キャッシュパスにヒット、そのまま実行、計算不要

# 失敗パス：自動学習
# → fail_count >= 3 → 自動的にブラックリストに追加
# → 次回は再計画、古いパスは利用しない
```

### 戦略の自動蓄積

```python
# 操作 Agent が実行に成功した後、自動記録
memory.record_strategy(
    task_type="mine",
    description="まず松明を置いてから採掘",
    action_sequence=[...],
    success=True,
    context_tags=["nighttime", "cave"],
)

# 次回同じタスクタイプで最適な戦略を自動マッチング
best = memory.get_best_strategy("mine", context_tags=["nighttime"])
# → 成功率が最も高い戦略を返す
```

## 🛤️ インテリジェントナビゲーション

### ナビゲーションフロー

```
SmartNavigator.goto(x, y, z)
  │
  ├── 1. セキュリティチェック
  │     └── 目標が保護区域内？→ 警告するが拒否はしない
  │
  ├── 2. キャッシュパス照会
  │     └── 信頼できるキャッシュあり？→ キャッシュパスをそのまま実行
  │
  ├── 3. ナビゲーションコンテキスト取得
  │     ├── 除外区域（建築保護区域）
  │     ├── 危険エリア（溶岩、崖）
  │     └── 信頼できるパス参照
  │
  ├── 4. Baritone 経路探索（優先）
  │     ├── exclusion_zones 注入
  │     ├── 自動掘削 / 架橋 / 泳行
  │     └── 落下コスト / 溶岩回避
  │
  ├── 5. A* 経路探索（フォールバック）
  │     └── 基本グリッド A* + ブロック通行可能性判定
  │
  └── 6. 結果記録
        ├── 成功 → cache_path(success=True)
        └── 失敗 → cache_path(success=False) + ブラックリスト登録の可能性
```

### Baritone 統合

| 特性 | Baritone | 基本 A* |
|------|----------|---------|
| 経路探索アルゴリズム | 改良 A* + コストヒューリスティック | 標準 A* |
| 掘削 | ✅ 障害物を自動掘削 | ❌ |
| 架橋 | ✅ scaffold モード | ❌ |
| 泳行 | ✅ | ❌ |
| 垂直移動 | ✅ ジャンプ/はしご/蔦 | ⚠️ 1ブロックのみ |
| 溶岩回避 | ✅ コストペナルティ | ❌ |
| 落下コスト | ✅ ヒューリスティック関数に算入 | ❌ |
| 除外区域 | ✅ `exclusionAreas` | ❌ |
| **建築保護** | ✅ `no_break` 区域を注入 | ❌ |

### 除外区域タイプ

| タイプ | 説明 | 由来 |
|--------|------|------|
| `no_break` | ブロック破壊を禁止 | 建築保護区域、基地 |
| `no_place` | ブロック設置を禁止 | 建築保護区域 |
| `avoid` | 完全に迂回 | 危険エリア（溶岩等） |

## 🤖 デュアル Agent アーキテクチャ

### なぜデュアル Agent が必要なのか？

```
単一 Agent の問題：
  チャットコンテキスト + 操作コンテキスト → Token 爆発（>4000/回）
  操作失敗がチャットを汚染 → 会話体験が低下
  毎回の操作に完全なチャット履歴が必要 → 無駄

デュアル Agent 方案：
  メイン Agent：チャットのみ、スライディングウィンドウ 20 件、約50 Token/回
  操作 Agent：ステートレス、毎回全新的コンテキスト、<1500 Token/回
```

### フロー

```
プレイヤーのメッセージ
  → メイン Agent チャット（永続コンテキスト）
  → [TASK:xxx] タグを識別
  → タスク記述を抽出
  → 操作 Agent 実行（ステートレス）：
      ├── Skill マッチング照会
      ├── 記憶コンテキスト注入
      ├── L1/L2: キャッシュ Skill を実行
      ├── L3: AI がテンプレートを埋めて実行
      └── L4: AI が完全に推論して実行
  → メイン Agent がフォーマットして応答 → プレイヤー
```

```bash

git clone https://github.com/bmbxwbh/BlockMind.git
cd BlockMind

# ワンクリック起動（依存関係自動インストール + MC サーバー + BlockMind + WebUI）
```

## 📝 Skill DSL システム

### タスク分類

| レベル | タイプ | 例 | キャッシュ戦略 |
|--------|--------|------|----------------|
| L1 | 固定タスク | "家に帰る" | そのまま実行 |
| L2 | パラメータ化タスク | "ダイヤモンドを10個掘る" | パラメータ付きキャッシュ |
| L3 | テンプレートタスク | "避難所を建設" | テンプレートマッチング |
| L4 | 動的タスク | "エンダードラゴンを倒すのを手伝って" | AI 推論 |

### Skill YAML 例

```yaml
skill_id: mine_diamonds
name: "ダイヤモンド採掘"
level: L2
parameters:
  count: {type: int, default: 10, min: 1, max: 64}
steps:
  - action: pathfind
    target: {y: -59}
    note: "ダイヤモンド層へ移動"
  - action: dig_loop
    block: diamond_ore
    count: ${count}
  - action: pathfind
    target: home
    note: "基地へ帰還"
```

## ❓ FAQ

**Q: Baritone のインストールは必須ですか？**
A: いいえ、必須ではありません。Baritone はオプションの依存関係で、未インストールの場合は基本 A* 直線移動にフォールバックします。

**Q: 記憶データはどこに保存されますか？**
A: `data/memory/` ディレクトリ配下の 5 つの JSON ファイルに、セッションを跨いで保持されます。

**Q: 建築保護はどのように有効になりますか？**
A: 2つの方法があります：① 手動登録 ② 自動検知（60秒ごとにスキャン）。

**Q: どの AI プロバイダーをサポートしていますか？**
A: OpenAI 互換フォーマット（DeepSeek/OpenRouter/MiMo 等を含む）+ Anthropic フォーマット。

**Q: Docker イメージのサイズは？**
A: 約 200MB、python:3.11-slim ベースのマルチステージビルドです。
